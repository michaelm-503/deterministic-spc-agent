
# Planner Schema — Deterministic SPC Agent

The planner schema defines the structure of JSON execution plans.

LLMs generate plans that conform to this schema.

Plans are validated before execution.

---

# Plan Types

The system supports two plan types:

| Plan Type            | Description                                              |
|----------------------|----------------------------------------------------------|
| Execution Plan       | Executes SQL, preprocessing, and outputs                 |
| Replot Plan          | Generates new visualizations from existing run artifacts |

Execution plans are typically generated by the LLM planner.

Replot plans may be generated by the planner or manually authored for visualization adjustments.

---

# Plan Library

A plan library contains multiple runs.

Example:

```
{
  "runs": [
    { ... },
    { ... }
  ]
}
```

---

# Run Plan

Each run defines a batch of jobs.

```
{
  "run_id": "example_run",
  "jobs": [ ... ]
}
```

---

# Job Structure

Each job describes one analytics workflow, including SQL template, preprocess script, filters, and params. A job may generate zero or more output artifacts (plots or tables).

```
{
  "job_id": "CPR11_temperature_motor",
  "sql_template": "entity_sensor_history",
  "preprocess": "ewma_spc",
  "filters": { ... },
  "params": { ... },
  "outputs": { ... }
}
```

---

### SQL Template

SQL templates define how data is extracted. Only registered templates may execute.



Templates are stored under: `sql/`

Example: `entity_sensor_history`

---

###	 Preprocess

Preprocess modules apply deterministic transformations. Only registered scripts may execute.

Job parameters are typically applied to the preprocess module.

Examples of preprocessing transformations:

- EWMA smoothing
- SPC violation detection

---

### Filters

| Field           | Required  | Description                                     |
|-----------------|-----------|-------------------------------------------------|
| entity_group    | ✅         | Required for fleet-level views                  |
| entity          | Depends   | Required for single-entity views                |
| sensor          | ✅         | Required for sensor-based views                 |
| start_ts        | ❌        | ISO datetime; SQL-level filter (optional)       |
| end_ts          | ❌        | ISO datetime; SQL-level filter (optional)       |

Note: Some plot types require specific filters (e.g., spc_time_series requires entity).

---

### Params (Job)

| Field           | Required  | Description                                     |
|-----------------|-----------|-------------------------------------------------|
| ewma_alpha      | ❌        | Sets EWMA alpha parameter. Default value = 0.2  |

---

# Parameter Hierarchy

Parameters may appear at two levels:

| Level | Applies To |
|------|-------------|
| Job-level params | Preprocessing logic |
| Output-level params | Plot or table behavior |

Output parameters override visualization defaults but do not affect preprocessing.

# Outputs

Outputs specify plots and summarized data tables. Only registered scripts may execute.

Output objects can have defined parameters that are applied on a per object basis.

### Plot Specification

```
"outputs": {
  "plots": [
    {
      "plot": "spc_time_series",
      "plot_name": "plot.png",
	  "params": {}
    }
  ]
}
```

### Supported Plot-Level Parameters

All parameters are optional and will default to settings below, as indicated by chart type.

| Parameter   | Type     | Description                           | Entity Time Trend | Fleet Time Trend | Boxplot |
|-------------|----------|---------------------------------------|-------------------|------------------|---------|
| show_raw    | bool     | Show raw scatter points               | true              | false            | n/a     |
| show_ewma   | bool     | Show EWMA line                        | true              | true             | false   |
| show_limits | bool     | Show SPC limits                       | true              | true             | true    |
| legend      | bool     | Show legend                           | true              | true             | false   |
| window_days | int      | Slice last N days of dataset          | null              | null             | null    |
| x_min       | datetime | Override x-axis lower bound           | null              | null             | n/a     |
| x_max       | datetime | Override x-axis upper bound           | null              | null             | n/a     |
| y_min       | float    | Override y-axis lower bound           | null              | null             | null    |
| y_max       | float    | Override y-axis upper bound           | null              | null             | null    |
| entities    | list     | Entity subset on fleet charts         | n/a               | [ *all* ]        | [ *all* ] |

Notes:
- Violations are shown only if show_limits=true.
- Limits are drawn using the latest non-null limit values.
- Violations are computed from raw values vs. limits (null-safe).
- `window_days` and `entities` are applied after SQL + preprocessing and before each plot call
- `x_min`/`x_max` is applied to plot axis and do not affect data slicing.
	
### Table Specification

```
"outputs": {
  "tables": [
	{
	  "table": "fleet_ooc_summary",
	  "table_name": "summary.csv",
	  "params": {}
	}
  ]
}
```

### Supported Table-Level Parameters

| Parameter   | Type     | Default | Description                           |
|-------------|----------|---------|---------------------------------------|
| window_days | int      | null    | Slice last N days of dataset          |
| start_ts    | datetime | null    | Explicit start bound                  |
| end_ts      | datetime | null    | Explicit end bound                    |
| entities    | list     | [all]   | Entity subset on fleet charts         |

Notes:
- Tables must write a CSV artifact.
- Time slicing and `entities` are applied after SQL + preprocessing and before each table call.
- Precedence: if `start_ts` or `end_ts` is provided, those bounds are applied; otherwise `window_days` is applied (if present).
- `start_ts` or `end_ts` flags passed as a table parameter will only affect the associated table. These flags do not apply to the job-level SQL filter.

---

# Replot Plans

Replot plans reference a previous run.

```
{
  "mode": "replot",
  "run_dir": "runs/<timestamp>",
  "jobs": [ ... ]
}
```

Replots regenerate visual outputs using existing `processed_data.csv` artifacts.

SQL execution and preprocessing are not re-run.

Replots create a new artifact directory under: `job/replots/<timestamp>/`

---

# Planner Guidelines

When generating plans, the LLM should:

- Prefer minimal overrides
- Use defaults when possible
- Avoid specifying parameters unless required
- Generate one job per analytic question
- Prefer entity-level plots unless fleet-level comparison is requested

The planner must not:

- invent SQL templates
- invent preprocess modules
- invent plot types

---

# Execution Rules

1.  A plan library may contain multiple run plans. 
2.  A run plan may contain multiple jobs.
3.  SQL is executed once per job.
4.  Preprocessing is executed once per job.
5.  Output (zero or more, plots or tables) reuse the same processed dataset.
6.  Filename collisions are resolved automatically by the execution engine.
7.  Defaults are defined in code, not in the JSON schema.
8.  Replot is additive and generates additional outputs for a specified job.

---

# Registry Contract

Valid keys must exist in:
- `SQL_REGISTRY`
- `PREPROCESS_REGISTRY`
- `PLOT_REGISTRY`
- `TABLE_REGISTRY`

Invalid keys result in execution error.

# Minimal Valid Example

``` json
{
  "runs": [
    {
      "run_id": "example",
      "request_text": "Show vibration trend for CNC01",
      "jobs": [
        {
          "job_id": "cnc01_vibration",
          "sql_template": "entity_sensor_history",
          "preprocess": "ewma_spc",
          "filters": {
            "entity_group": "CNC",
            "entity": "CNC01",
            "sensor": "vibration_rms",
            "start_ts": null,
            "end_ts": null
          },
          "outputs": {
            "plots": [
              {
                "plot": "spc_time_series",
                "plot_name": "cnc01_vibration.png"
              }
            ]
          }
        }
      ]
    }
  ]
}
```
